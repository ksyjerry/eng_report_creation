## 2025-03-21: HYBE 2025 영문재무제표 생성

### 발견 1: Cross-language row matching 실패 (매칭률 0%)

**상황**: DSD 한국어 라벨과 DOCX 영어 라벨을 직접 비교
**증상**: 매칭률이 0%에 가까움 (언어가 달라서 문자열이 전혀 안 맞음)
**해결**: IFRS 용어 사전으로 한국어→영어 번역 후 비교하여 매칭률 67.4% 달성
**교훈**: 교차언어 매칭은 반드시 번역 후 비교해야 함

### 발견 2: Column index 밀림 (spacer column)

**상황**: HYBE DOCX에 spacer column 존재 (너비 58 dxa)
**증상**: 값이 한 칸씩 밀려서 spacer column에 들어감
**해결**: logical→physical column 변환 적용
**교훈**: 테이블을 처음 만나면 반드시 column info 확인

### 발견 3: Year cascade

**상황**: "2024"→"2025", "2023"→"2024" 순차 교체
**증상**: 모든 연도가 "2025"로 됨
**해결**: atomic 교체로 변경 (한 번에 모든 교체 적용)
**교훈**: 연도 교체는 절대 순차적으로 하지 말 것

### 발견 4: Split year in headers

**상황**: Header의 "2024"가 "202"+"4"로 Run 분할
**증상**: 연도 교체가 안 됨 (단순 검색으로 "2024" 못 찾음)
**해결**: cross-run 교체 적용
**교훈**: 연도 교체 후 반드시 search로 잔존 확인

### 발견 5: 유사 라벨 잘못된 매칭

**상황**: "평가이익(Gain on valuation)"과 "평가손실(Loss on valuation)"
**증상**: IFRS 번역 후 유사도가 높아서 잘못 매칭됨
**해결**: 값 기반 검증 추가 (DSD 전기값 ≈ DOCX 당기값 확인)
**교훈**: 라벨 매칭은 반드시 값 교차검증과 함께

### 발견 6: 빈 셀의 numeric 판정

**상황**: 빈 셀("")이 _is_numeric_text()에서 True 반환
**증상**: spacer column이 데이터 열로 감지됨
**해결**: 실제 숫자값이 있는 셀만 numeric으로 판정
**교훈**: 빈 셀, 대시, 백슬래시는 숫자에서 제외

### 발견 7: gridSpan이 spacer를 포함

**상황**: 테이블 헤더 "2024"가 gridSpan=6으로 spacer+data column 모두 포함
**증상**: 기간 열이 spacer column으로 감지됨
**해결**: 실제 데이터 행에서 숫자가 있는 열로 보정
**교훈**: 헤더의 기간 열은 데이터 행으로 검증해야 함
