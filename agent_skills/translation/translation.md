# 번역 (Korean → English)

## 핵심 원칙

**전기 DOCX의 영문이 최우선. 번역은 최소한으로.**

DOCX 템플릿 = 전기 영문 보고서이다. 감사인이 이미 검토한 완성된 번역이 들어있다.
당기 DSD의 한국어 대부분은 전기와 동일하거나 약간 수정된 것이므로,
번역 = "새로 번역" 이 아니라 **"전기 영문 재사용 + 변경분만 최소 조정"** 이다.

## 번역 4-Tier 우선순위

### Tier 1: 전기 영문 그대로 (Exact Reuse) — 대부분(80%+)

한국어가 전기와 완전히 동일 → DOCX의 영문을 그대로 사용.
LLM 호출 없음. 가장 정확하고 빠르다.

```
예:
  DSD 전기: "유동자산"  → DOCX: "Current assets"
  DSD 당기: "유동자산"  → 그대로 "Current assets" 사용
```

**식별 방법**: 값 매칭으로 같은 행 확인 → 한국어 라벨 비교 → 동일하면 Tier 1

### Tier 2: 전기 영문 기반 최소 조정 (Minimal Adjust) — 일부

한국어가 약간 변경됨 → 전기 영문을 기반으로 변경된 부분만 조정.

```
예:
  전기 한국어: "기타포괄손익-공정가치금융자산"
  전기 영문:   "Financial assets at FVOCI"
  당기 한국어: "기타포괄손익-공정가치측정금융자산"  (측정 추가)
  → LLM에게 전기 영문 + 변경 내용 제공 → "Financial assets measured at FVOCI"
```

**LLM 프롬프트 핵심**: "전기 영문을 최소한으로만 수정하세요. 스타일과 용어를 유지하세요."

### Tier 3: 전기 문체 맞춤 신규 번역 (Style-Matched New) — 소수

완전히 새로운 항목 → 주변 전기 영문의 문체/어투에 맞게 번역.

```
예:
  새 계정과목이 추가됨 (전기에 없음)
  → 같은 테이블의 다른 행 영문들을 style sample로 LLM에 제공
  → "이 테이블의 다른 항목들처럼 번역하세요"
```

**LLM 프롬프트 핵심**: 전후 맥락 영문 5~10줄을 함께 제공. "이 문체에 맞춰 번역하세요."

### Tier 4: IFRS 용어 기반 번역 (Fallback) — 극소수

맥락이 전혀 없는 경우 → IFRS 표준 용어집 + 일반 LLM 번역.

## Glossary 자동 구축

### 방법: 값 매칭 → 라벨 추출

```
1. DSD 전기 값과 DOCX 당기 값으로 행 매칭 (숫자 기반)
2. 매칭된 행에서:
   - DSD 라벨 (한국어) = glossary key
   - DOCX 라벨 (영어) = glossary value
3. 모든 테이블에서 반복 → glossary 누적 확장
```

**이 Glossary가 가장 신뢰도 높은 이유**:
- 같은 문서 쌍에서 추출 → 이 회사의 실제 사용 표현
- 감사인이 이미 검토한 번역
- 회사별 고유 표현 반영 (예: 같은 계정이라도 회사마다 영문 표현이 다름)

### Glossary 활용

```
테이블 1 처리 → glossary에 30개 쌍 추가
테이블 2 처리 → glossary에 20개 쌍 추가 (이미 있으면 중복 제거)
...
주석 처리 시 → 이미 60개 쌍이 축적되어 있으므로 대부분 Tier 1로 해결
```

## 주석 번역 특수 처리

주석은 긴 문단으로 구성. 문단 단위 비교가 핵심.

### 문단 매칭 전략

```
DSD 주석 12 문단들:         DOCX 주석 12 문단들:
[1] 가. 토지              ↔ [1] (a) Land
[2] 당기 중 변동...        ↔ [2] Changes during the period...
[3] (새로 추가된 문단)      ↔ (없음) → Tier 3 번역
[4] 나. 건물              ↔ [3] (b) Buildings
```

1. **순서 기반 매칭**: 대부분 같은 순서
2. **내용 기반 매칭**: 순서가 다르면 유사도로 매칭
3. **매칭된 문단**: 전기 영문 vs 당기 한국어 비교
   - 내용 동일 → 전기 영문 그대로 (Tier 1)
   - 약간 수정 → 전기 영문 기반 조정 (Tier 2)
4. **새 문단**: 전후 문단의 영문을 style sample로 제공 (Tier 3)

### 주석 내 테이블

주석 안의 테이블도 동일 원칙:
- 전기 테이블과 같은 구조 → 라벨 재사용
- 행이 추가됨 → 주변 행의 영문 스타일에 맞춰 번역

## 문체 일관성 규칙

### 같은 문서 내 통일

```
나쁜 예:
  주석 5: "The Group recognized a gain of..."
  주석 8: "A gain was recognized by the Company..."
  → 주어, 시제, 표현이 불일관

좋은 예:
  주석 5: "The Group recognized a gain of..."
  주석 8: "The Group recognized an impairment loss of..."
  → 같은 패턴
```

### 전기 문체 계승

전기 보고서가 사용하는 패턴을 파악하고 따라야 함:
- 주어: "The Group" vs "The Company" vs "The consolidated entity"
- 시제: 과거형 위주 vs 현재완료 혼용
- 구조: 능동태 vs 수동태 경향
- 약어: "FVTPL" vs "fair value through profit or loss" (전기 사용 패턴 따름)

## 번역하지 않는 것

| 대상 | 처리 |
|------|------|
| 숫자 | 형식만 변환 (number_format.md 참조) |
| 날짜 | 형식만 변환: "2025년 12월 31일" → "December 31, 2025" |
| 회사명 | DOCX에 이미 있는 영문명 사용 |
| 단위 | DOCX 기존 표현: "(In thousands of Korean won)" |

## 번역 일관성 검증

번역 완료 후 확인할 것:
1. 같은 한국어 → 같은 영어인지 (문서 전체 검색)
2. 전기에 있던 표현이 불필요하게 바뀌지 않았는지
3. 새 번역의 문체가 주변과 이질적이지 않은지

## IFRS 표준 용어

`ifrs_terms.md` 참조. 단, IFRS 용어보다 **전기 DOCX의 실제 표현**이 우선.
(회사별로 IFRS 표준 용어와 다른 표현을 쓰는 경우가 있음)
