# 숫자 형식 (Number Formatting)

## 개요
DSD의 숫자를 DOCX에 넣을 때, DOCX의 기존 형식에 맞춰야 한다.
숫자 **값**은 절대 변환하지 않고, **표현 형식**만 맞춘다.

## 형식 대응표

### 양수
| DSD | DOCX | 비고 |
|-----|------|------|
| 1,234,567 | 1,234,567 | 천단위 콤마 (동일) |
| 1234567 | 1,234,567 | 콤마 추가 필요할 수 있음 |

### 음수
| DSD | DOCX | 비고 |
|-----|------|------|
| -1,234,567 | (1,234,567) | **괄호가 표준** |
| (1,234,567) | (1,234,567) | 동일 |
| △1,234,567 | (1,234,567) | 삼각형 → 괄호 |

### 0 (제로)
| DSD | DOCX | 비고 |
|-----|------|------|
| 0 | - | **대시가 일반적** |
| - | - | 동일 |
| — | - | em dash → en dash |

### 빈 값
| DSD | DOCX | 비고 |
|-----|------|------|
| (비어있음) | (비어있음) | 변경하지 않음 |
| \ | \ | 구분선 (spacer) |

## 형식 감지

DOCX 기존 형식을 감지하는 방법:

1. 같은 열의 다른 셀들을 확인
2. 양수 형식: 천단위 콤마 사용 여부
3. 음수 형식: 괄호 vs 마이너스
4. 0 표현: 대시 vs "0" vs 빈칸

```
예시:
  col 2 기존 셀들: "1,234,567", "(456,789)", "-"
  → 양수: 천단위 콤마 ✓
  → 음수: 괄호 표현 ✓
  → 0: 대시 ✓
```

## 한국어 숫자 파싱

```python
"1,234,567"     → 1234567
"(1,234,567)"   → -1234567
"-1,234,567"    → -1234567
"△1,234,567"    → -1234567
"-"             → 0
"—"             → 0
"–"             → 0
""              → None (파싱 불가)
"\"             → None (spacer, 숫자 아님)
"₩1,234,567"   → 1234567 (통화기호 제거)
```

## 영어 숫자 포맷팅

```python
1234567   → "1,234,567"
-1234567  → "(1,234,567)"
0         → "-"
```

## 단위 주의

DSD와 DOCX의 단위가 다를 수 있다:

| DSD 단위 | DOCX 단위 | 변환 |
|---------|----------|------|
| 원 | 원 | 그대로 |
| 원 | 천원 | ÷ 1,000 |
| 천원 | 천원 | 그대로 |
| 백만원 | 천원 | × 1,000 |

**중요**: 단위 정보는 테이블 상단이나 주석 첫 부분에 표시됨
- DSD: "(단위: 원)", "(단위: 천원)"
- DOCX: "(In Korean won)", "(In thousands of Korean won)"

단위 변환이 필요한 경우 반드시 확인 후 적용.
잘못된 단위 변환은 모든 숫자가 1,000배 틀리는 결과를 초래한다.

## 소수점

- IFRS 재무제표에서 소수점은 드묾
- 주당이익(EPS) 등에서 가끔 사용: "1,234.56"
- 한국: 소수점 "." 사용 (영어와 동일)

- [ ] 포맷 변환 후 값이 변하지 않았는지 확인
