# 검증 (Verification)

## 개요
모든 변경 적용 후, 결과물이 올바른지 전수 검증한다.
검증은 **DSD 원본이 절대적인 기준(ground truth)**이다.

## 검증 체크리스트

### 1. 숫자 정합성 (가장 중요)

DSD의 모든 숫자가 출력 DOCX에 올바르게 반영되었는지 확인.

```
검증 방법:
  DSD 테이블의 각 행에 대해:
    1. 매칭된 DOCX 행을 찾음
    2. DSD 당기값 == DOCX 새 당기값 인지 확인
    3. DSD 전기값 == DOCX 새 전기값 인지 확인

  허용 오차: 절대값 차이 <= 1 (반올림)
```

**검증 순서**:
1. 재무상태표 (BS) — 가장 중요
2. 포괄손익계산서 (IS)
3. 현금흐름표 (CF)
4. 자본변동표 (CE) — 구조가 복잡하므로 주의
5. 주석별 테이블

### 2. 완전성 (Completeness)

```
확인 사항:
  - [ ] DSD의 모든 주석이 DOCX에 존재하는지
  - [ ] DSD의 모든 테이블이 DOCX에 반영되었는지
  - [ ] 누락된 행이 없는지 (DSD에 있는데 DOCX에 없는 행)
  - [ ] 재무제표 4개 모두 존재하는지
```

### 3. 연도 일관성 (Year Consistency)

```
확인 사항:
  - [ ] Header/Footer에 올바른 연도만 존재
  - [ ] 테이블 헤더에 올바른 연도만 존재
  - [ ] 본문에 이전 연도가 남아있지 않은지
  - [ ] 예외: 설립연도, 법률 참조 등은 이전 연도가 정상

방법:
  search_text("2023")  → 전기DOCX의 전기연도가 남아있으면 문제
  search_text("2024")  → 전기DOCX의 당기연도가 적절한 곳에만 있는지
  search_text("2025")  → 새 당기연도가 올바르게 들어갔는지
```

### 4. 서식 보존 (Format Preservation)

```
확인 사항:
  - [ ] DOCX가 정상적으로 열리는지
  - [ ] 테이블 구조 (열 수, 병합)가 원본과 동일한지
  - [ ] 폰트, 크기, 정렬이 유지되었는지
  - [ ] 페이지 나누기가 올바른 위치에 있는지
```

### 5. 번역 품질

```
확인 사항:
  - [ ] 한국어가 남아있지 않은지 (번역 누락)
  - [ ] 용어 일관성 (같은 한국어 = 같은 영어)
  - [ ] IFRS 표준 용어 사용 여부
  - [ ] "[NEEDS_TRANSLATION: ...]" 마커가 남아있지 않은지
```

## 교차 검증 (Cross-Check)

재무제표 간의 논리적 일관성을 확인:

### BS 균형
```
자산총계 == 부채총계 + 자본총계

확인: Total assets == Total liabilities + Total equity
허용 오차: 1 (반올림)
```

### IS 일관성
```
매출총이익 == 매출액 - 매출원가
영업이익 == 매출총이익 - 판매비와관리비
당기순이익 == 법인세비용차감전순이익 - 법인세비용
```

### CF 균형
```
기말현금 == 기초현금 + 영업CF + 투자CF + 재무CF + 환율효과

확인: Cash at end == Cash at beginning + Operating + Investing + Financing
```

### BS-IS 연결
```
이익잉여금 변동 ≈ 당기순이익 - 배당금
(정확하지 않을 수 있음 — 기타 변동 항목 존재)
```

## 오류 심각도

| 심각도 | 설명 | 대응 |
|--------|------|------|
| CRITICAL | 숫자가 틀림, 재무제표 균형 불일치 | 반드시 수정 |
| WARNING | 번역 누락, 연도 잔존, 행 누락 | 가능하면 수정 |
| INFO | 사소한 형식 차이, 서식 미세 변화 | 기록만 |

## 검증 실패 시

1. **숫자 오류**: 해당 셀을 DSD 원본값으로 직접 덮어쓰기
2. **연도 오류**: 해당 부분에 연도 롤링 재적용
3. **번역 누락**: LLM으로 재번역
4. **서식 깨짐**: 해당 부분을 원본에서 다시 clone
5. **3번 시도해도 해결 안 됨**: 에스컬레이션 (사람에게 보고)

## 검증 보고서

최종 검증 결과를 구조화된 보고서로 출력:

```
== 검증 보고서 ==
전체 상태: PASS / FAIL

[숫자 정합성]
  BS: 45행 중 45행 일치 ✓
  IS: 38행 중 38행 일치 ✓
  CF: 52행 중 51행 일치 (1행 불일치) ✗
    - CF 행 23: DSD 789,012 vs DOCX 789,011 (차이: 1, 반올림)
  주석: 검증 완료 (오류 0건)

[완전성]
  주석: DSD 45개 중 43개 반영, 2개 신규 추가 필요
  재무제표: 4개 모두 존재 ✓

[연도 일관성]
  "2023" 잔존: 0건 ✓
  "2024" 잔존: 올바른 위치에만 존재 ✓

[서식 보존]
  DOCX 정상 열기: ✓
  테이블 구조 유지: ✓

CRITICAL: 0건
WARNING: 2건 (신규 주석 미반영)
INFO: 1건 (반올림 차이)
```
