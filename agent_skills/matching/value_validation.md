# 값 기반 검증 (Value Validation)

## 핵심 원리

**DSD 전기값 == DOCX 당기값**

이것이 행 매칭의 가장 강력한 검증 수단이다.

### 왜 이것이 성립하는가?

- DOCX는 **전기** 영문 재무제표이다
- DOCX의 당기 열 = 전기의 당기 = **현재 DSD의 전기**
- 따라서 DSD 전기값과 DOCX 당기값이 같으면 같은 행이다

```
시간축:

           전기 DOCX의 전기열    전기 DOCX의 당기열    DSD의 전기열    DSD의 당기열
           (2년 전 값)           (1년 전 값)           (1년 전 값)     (올해 값)
                                 ↕ 같아야 함            ↕
유동자산:     900,000             1,200,000             1,200,000      1,500,000
```

## 활용 방법

### 1. 행 매칭 검증

라벨로 매칭한 후, 값으로 교차검증:

```
라벨 매칭: DSD "감가상각비" → DOCX "Depreciation" (유사도 0.7)
값 검증: DSD 전기 500,000 vs DOCX 당기 500,000 → 일치! ✓ 매칭 확정

라벨 매칭: DSD "무형자산상각비" → DOCX "Depreciation" (유사도 0.6)
값 검증: DSD 전기 50,000 vs DOCX 당기 500,000 → 불일치! ✗ 매칭 거부
```

### 2. 직접 매칭 (라벨 없이)

값이 유일하면 라벨 없이도 매칭 가능:

```
DSD 전기값 1,234,567 → DOCX에서 당기 열에 1,234,567이 있는 행 검색
→ 유일하게 하나 있으면 → 매칭
→ 여러 개 있으면 → 라벨과 함께 판단
```

### 3. 업데이트 후 검증

값을 쓴 후 올바르게 들어갔는지 확인:

```
DSD 당기값 1,500,000을 DOCX에 씀
→ read_cell()로 확인
→ "1,500,000" ✓
```

## 숫자 비교 방법

### 파싱
- "1,234,567" → 1234567
- "(1,234,567)" → -1234567
- "-" / "—" / "–" → 0
- "" (빈칸) → 비교 대상에서 제외
- "\" (백슬래시) → 비교 대상에서 제외

### 허용 오차
- 정수 비교: 절대차이 <= 1 (반올림 차이 허용)
- 비율 비교: abs(a - b) / max(abs(a), abs(b)) < 0.001 (0.1% 이내)

### 매우 작은 값
- 양쪽 다 절대값 < 1이면 비교 무의미 → 무시

### 값 비율 검증 (잘못된 매칭 감지)
```
ratio = max(abs(dsd), abs(docx)) / min(abs(dsd), abs(docx))
if ratio > 2.0:
    → 잘못된 매칭 가능성 높음 → 매칭 거부 또는 에스컬레이션
```

## 주의사항

1. **0과 대시는 매칭에 쓰지 않는다**
   - 너무 많은 행이 0이나 대시를 가지고 있어 매칭이 모호해짐

2. **단위 차이 주의**
   - DSD가 "원" 단위, DOCX가 "천원" 단위일 수 있음
   - → 단위(unit) 정보를 먼저 확인

3. **음수 표현 차이**
   - DSD: -1,234 또는 △1,234
   - DOCX: (1,234)
   - → 절대값으로 비교 후 부호 확인

4. **열 위치 확인 필수**
   - DSD의 "당기" 열 인덱스와 DOCX의 "당기" 열 인덱스가 다를 수 있음
   - → 기간 열 감지 후 비교 (table_matching.md 참조)
