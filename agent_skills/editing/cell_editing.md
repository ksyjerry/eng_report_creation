# 셀 값 편집

## 개요
테이블 셀의 텍스트를 변경하면서 기존 Run 서식을 완벽하게 보존하는 방법.

## 셀의 XML 구조

```xml
<w:tc>
  <w:tcPr>...</w:tcPr>         ← 셀 속성 (테두리, 배경 등) — 건드리지 않음
  <w:p>                         ← 문단
    <w:pPr>...</w:pPr>          ← 문단 속성 (정렬 등) — 건드리지 않음
    <w:r>                       ← Run 1
      <w:rPr>                   ← Run 속성 (폰트, 크기, 볼드 등) — 보존해야 함
        <w:rFonts w:ascii="Times New Roman"/>
        <w:sz w:val="18"/>
        <w:b/>                  ← 볼드
      </w:rPr>
      <w:t>1,234,567</w:t>     ← 텍스트 — 이것만 교체
    </w:r>
    <w:r>                       ← Run 2 (있을 수도 없을 수도)
      <w:rPr>...</w:rPr>
      <w:t></w:t>
    </w:r>
  </w:p>
</w:tc>
```

## 텍스트 교체 방법

### 기본 전략: "마지막 Run에 넣고, 나머지 비우기"

1. 셀의 모든 Run을 수집
2. 텍스트가 있는 **마지막 Run**을 찾음
3. 그 Run의 첫 번째 `<w:t>`에 새 텍스트 삽입
4. 같은 Run의 나머지 `<w:t>`를 빈 문자열로
5. 다른 모든 Run의 `<w:t>`도 빈 문자열로

### 왜 "마지막 Run"인가?

재무제표에서 셀의 마지막 Run이 보통 실제 숫자를 담고 있다.
앞쪽 Run은 들여쓰기용 공백이나 통화 기호인 경우가 많다.

### Run이 없는 경우

셀에 Run이 하나도 없으면 (빈 셀에 값을 넣을 때):
1. 같은 행의 다른 셀에서 Run을 찾음
2. 그 Run의 `<w:rPr>`을 deepcopy
3. 새 `<w:r>` 생성: deepcopy한 rPr + 새 `<w:t>`
4. 셀의 첫 `<w:p>`에 삽입

## 서식 보존 체크리스트

변경 후 다음이 유지되어야 한다:

- [ ] 폰트 (rFonts)
- [ ] 크기 (sz)
- [ ] 볼드 (b)
- [ ] 이탤릭 (i)
- [ ] 색상 (color)
- [ ] 정렬 (jc) — 셀의 pPr에 있음
- [ ] 들여쓰기 (ind) — 라벨 셀의 계층 구조
- [ ] 테두리 (tcBorders) — 합계 행의 하단 선 등
- [ ] xml:space="preserve" — 모든 `<w:t>`에 설정

## 숫자 형식 맞추기

새 숫자를 넣을 때 DOCX 기존 형식을 따른다:

```
기존 셀: "(987,654)"  → 음수 = 괄호
새 값:   -1,234,567   → "(1,234,567)" 로 변환하여 삽입

기존 셀: "1,234,567"  → 양수 = 천단위 콤마
새 값:   2345678      → "2,345,678" 로 변환하여 삽입

기존 셀: "-"          → 0 = 대시
새 값:   0            → "-" 로 변환하여 삽입
```

**형식 감지 순서**:
1. 같은 열의 다른 셀에서 형식 패턴 확인
2. 양수: 천단위 콤마 여부
3. 음수: 괄호 vs 마이너스
4. 0: 대시 vs "0"

## 합계 행 주의

합계 행(Total)은 보통:
- 볼드 텍스트 (`<w:b/>`)
- 하단 테두리 (`<w:tcBorders>`)
- 때로 상단 테두리도

합계 셀의 값을 변경할 때 이 서식이 유지되는지 반드시 확인.

## 주의사항

1. **rPr 순서**: rPr은 Run의 첫 번째 자식이어야 함
2. **빈 Run 삭제 금지**: Run을 삭제하면 다른 서식 참조가 깨질 수 있음
   → 텍스트만 비우고 Run 구조는 유지
3. **여러 문단**: 하나의 셀에 여러 `<w:p>`가 있을 수 있음
   → 보통 첫 번째 문단만 사용
4. **xml:space**: 모든 `<w:t>`에 `xml:space="preserve"` 설정
   → 공백이 잘리지 않도록
