# 연도 롤링 (Year Rolling)

## 개요
전기 DOCX의 연도 참조(예: 2024/2023)를 당기(2025/2024)로 변경하는 작업.

## 적용 대상

모든 곳에 연도가 있을 수 있다. 빠짐없이 처리해야 한다:

- [ ] Header (머리글) — 모든 섹션
- [ ] Footer (바닥글) — 모든 섹션
- [ ] 테이블 헤더 행 (상위 1~2행)
- [ ] 본문 문단 (테이블 내부 제외)
- [ ] 표지 (cover page)

## 교체 규칙

전기 DOCX가 2024/2023이면:

| 원래 | 변경 후 |
|------|--------|
| 2024 | 2025 |
| 2023 | 2024 |

**교체 순서**: 큰 연도부터 먼저!
- 먼저: 2024 → 2025
- 다음: 2023 → 2024

## 치명적 오류: Cascade 문제

**절대 순차적으로 교체하지 마라.**

```
나쁜 예 (순차 교체):
  원본:  "December 31, 2024 and 2023"
  1단계: "2023" → "2024"  → "December 31, 2024 and 2024"
  2단계: "2024" → "2025"  → "December 31, 2025 and 2025"  ← 전부 2025!

올바른 예 (atomic 교체):
  원본 전체 텍스트를 한 번에 읽고
  모든 교체를 한 번에 적용:
    "2024" → "2025"
    "2023" → "2024"
  결과: "December 31, 2025 and 2024" ✓
```

**해결 방법**:
- `replace_in_headers_footers([("2024", "2025"), ("2023", "2024")])`
- 도구가 내부적으로 전체 텍스트를 연결 → 한 번에 교체 → 재분배

## Run 분할 문제

Word는 편집 이력에 따라 "2024"를 여러 Run으로 분할할 수 있다:

```xml
<w:r><w:t>December 31, 202</w:t></w:r>
<w:r><w:t>4</w:t></w:r>
```

이 경우 단순 텍스트 검색으로 "2024"를 찾을 수 없다!

**해결: Cross-Run 교체**

1. 문단 내 모든 `<w:t>` 텍스트를 연결: "December 31, 2024"
2. 연결된 텍스트에서 교체: "December 31, 2025"
3. 원래 `<w:t>` 요소 크기에 맞게 재분배:
   - 첫 `<w:t>`: "December 31, 202" (원래 길이만큼)
   - 둘째 `<w:t>`: "5" (나머지)

이 방법으로 Run 구조와 서식을 보존하면서 텍스트만 교체.

## 테이블 헤더의 연도

테이블 헤더 행에도 연도가 있다:

```
| 구분 | December 31, 2024 | December 31, 2023 |
```

- 상위 2행만 대상으로 교체 (데이터 행의 숫자를 건드리지 않도록)
- 같은 cross-run 교체 방법 사용

## 본문 문단의 연도

본문 텍스트에도 연도 참조가 있을 수 있다:

```
"For the year ended December 31, 2024, the Company..."
```

- 테이블 내부 셀은 제외 (테이블은 별도 처리)
- body 직속 `<w:p>`만 대상

## 검증 방법

교체 후 반드시:

1. `read_header_footer()` → 모든 header/footer에서 연도 확인
2. `search_text("2023")` → 이전 연도가 남아있지 않은지 (전기DOCX 기준)
3. `search_text("2025")`, `search_text("2024")` → 새 연도가 올바르게 들어갔는지

**예외**: 일부 텍스트에서 이전 연도가 정상적으로 남아있을 수 있음
- "Established in 2003" — 회사 설립연도
- "Act No. 12345 of 2022" — 법률 참조
- → 이런 경우는 변경하지 않아야 함
- → 현재 자동 교체는 모든 "2023"을 바꾸므로, 필요 시 사후 수정
